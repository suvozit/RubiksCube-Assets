Resource Page Description
This is a windows Dialog based application written in Visual C++/MFC that solves a scrambled Rubik's cube, showing the steps. A Rubik's cube configuration can be entered into the program using the 2D view. The 3D view was written using OpenGL.



The program solves each layer of the cube one at a time. It first solves the top layer, the middle layer, and then the bottom layer.

This page is a good example of this solution:
http://peter.stillhq.com/jasmine/rubikscubesolution.html

To minimize the number of moves for the user, the program solves the cube 6 different times, each time rotating the cube to use a different side as the top layer, and shows the solution with the least number of moves. But since it shows the solution to the user with the Blue side facing the user, the first layer that is being solved might not be the top layer. Feel free to comment on this as I have been debating on whether to solve the Yellow layer first all times or not.
